拿到（过滤/重排后的）top-k Node，怎么揉成一个答案？Response Synthesizer 决定“多个片段→单个回答”的策略，核心是应对上下文窗口与片段数量的权衡。Given the (filtered/reranked) top-k Nodes, how do you fuse them into one answer? The Response Synthesizer sets the “many chunks → one answer” strategy — chiefly trading off context window against number of chunks.

🧩 生活类比🧩 Analogy
写手拿到几页资料的几种写法：逐页精炼（refine）、塞满一次写（compact）、分组总结再合并（tree_summarize）、每页各答再汇总（accumulate）。
Different ways a writer drafts from several pages: refine page by page, pack as many as fit and write once (compact), summarize in groups then merge (tree_summarize), or answer each page then collect (accumulate).

from llama_index.core import get_response_synthesizer

# 方式一：直接在 query engine 上选模式
engine = index.as_query_engine(response_mode='tree_summarize')
print(engine.query('把这些文档总结成 3 点'))

# 方式二：显式造一个合成器（便于自定义装配，见下一课）
synth = get_response_synthesizer(response_mode='compact')
✅ 本课要点✅ Key points
  • Synthesizer 解决“多个 Node 如何合成一个答案”。The synthesizer solves “how many Nodes become one answer”.
  • compact 省调用，tree_summarize 擅总结，refine 重细读。compact saves calls, tree_summarize summarizes, refine reads carefully.
  • 模式选择本质是上下文窗口 vs 片段数量的权衡。Mode choice is fundamentally context window vs number of chunks.
💡 设计亮点💡 Design insight
把“拼接片段”这件容易写死的事抽象成可切换的 ResponseMode，让同一套检索结果能服务“精确问答”到“全局总结”不同需求。
Abstracting “stitch the chunks” into switchable ResponseModes lets one set of retrieved results serve everything from precise Q&A to global summarization.
