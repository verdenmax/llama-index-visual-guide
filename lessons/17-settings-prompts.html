全局配置 Settings 与 PromptSettings & Prompts

Settings 是全局默认配置（llm / embed_model / node_parser / chunk_size…），取代了旧的 ServiceContext。Prompt 模板则控制“到底怎么问 LLM”，可按领域定制语气与约束。Settings holds global defaults (llm / embed_model / node_parser / chunk_size…), replacing the old ServiceContext. Prompt templates control “how exactly we ask the LLM”, customizable per domain for tone and constraints.

🧩 生活类比🧩 Analogy
Settings 像项目的默认设置面板：定一次，处处生效，单处仍可覆盖。Prompt 模板像填空式提问公式，把检索到的上下文和用户问题填进固定结构。
Settings is the project's defaults panel: set once, applies everywhere, still overridable locally. A prompt template is a fill-in-the-blank question form that slots the retrieved context and the user's question into a fixed structure.

from llama_index.core import Settings, PromptTemplate
from llama_index.llms.openai import OpenAI

Settings.llm = OpenAI(model='gpt-4o-mini')   # 全局默认，一处生效
Settings.chunk_size = 512

qa = PromptTemplate(
    '只根据下面的上下文回答，未提及就说不知道，不要编造。\n'
    '上下文:\n{context_str}\n问题: {query_str}\n答案: '
)
engine = index.as_query_engine(text_qa_template=qa)   # 定制问答 Prompt
print(engine.query('退款政策？'))
✅ 本课要点✅ Key points
  • Settings 取代 ServiceContext，集中管理默认 llm/embed/切块等。Settings replaces ServiceContext, centralizing default llm/embed/chunking.
  • Prompt 模板用 {context_str} / {query_str} 注入上下文与问题。Prompt templates inject context and question via {context_str} / {query_str}.
  • 全局默认 + 局部覆盖：既省事又灵活。Global defaults + local overrides: convenient yet flexible.
💡 设计亮点💡 Design insight
“全局 Settings + 局部覆盖”避免了到处传配置对象的样板；而把 Prompt 显式成模板，让 RAG 的“最后一公里语气与约束”可控可审。
“Global Settings + local overrides” kills the boilerplate of threading a config object everywhere; making the prompt an explicit template keeps RAG's “last-mile tone and constraints” controllable and auditable.
