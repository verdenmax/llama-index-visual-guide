01 · LlamaIndex 与 RAG 是什么

🧪 自测 · 想一想为什么这么设计🧪 Self-check · think about the design

1. RAG（检索增强生成）的核心思想是？What is the core idea of RAG?
  1. 换一个更大的模型Switch to a bigger model
  2. 回答前先检索相关外部片段、塞进上下文再生成Retrieve relevant external snippets first, put them in context, then generate
  3. 把全部数据塞进一个超大提示词Stuff all data into one giant prompt
  4. 重新训练模型以记住新知识Retrain the model to memorize new knowledge
看答案与解析Show answer & explanation 点击展开expand
答案：BAnswer: B RAG = 先检索后生成：用检索到的相关片段作为上下文，让模型据此回答，避免重训成本与知识幻觉。RAG retrieves first, then generates: relevant snippets become context so the model answers from them, avoiding retraining cost and hallucination.
2. LlamaIndex 在 RAG 里主要负责什么？What does LlamaIndex mainly provide for RAG?
  1. 把加载→切块→索引→检索→合成这条管道标准化、可编排Standardizing and orchestrating the load→split→index→retrieve→synthesize pipeline
  2. 提供 GPU 推理Providing GPU inference
  3. 训练 embedding 模型Training embedding models
  4. 托管一个向量数据库服务Hosting a vector database service
看答案与解析Show answer & explanation 点击展开expand
答案：AAnswer: A LlamaIndex 是 RAG 的“数据与编排框架”：统一各阶段接口，底层换 LLM / 向量库 / embedding 都不改主链路。LlamaIndex is the data + orchestration framework for RAG: it unifies each stage so swapping the LLM / vector store / embedding doesn't change the main pipeline.
